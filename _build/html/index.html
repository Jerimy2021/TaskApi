Locust¶

class app.locustfile.WebsiteTestUser(*args, **kwargs)¶

A class that defines a user that will be simulated.

This class inherits from HttpUser, which is a class that defines a user that will be simulated.

This class has two tasks:

  • get_coordinates: This task sends a GET request to the /get_coordinates endpoint.

  • get_distance: This task sends a GET request to the /get_distance endpoint.

The wait_time attribute is set to a random value between 1 and 5 seconds. This means that the simulated user will wait between 1 and 5 seconds between each task.

abstract: bool = False¶

If abstract is True, the class is meant to be subclassed, and users will not choose this locust during a test

tasks: list[TaskSet | Callable] = [<function WebsiteTestUser.get_coordinates>, <function WebsiteTestUser.get_distance>]¶

Collection of python callables and/or TaskSet classes that the Locust user(s) will run.

If tasks is a list, the task to be performed will be picked randomly.

If tasks is a (callable,int) list of two-tuples, or a {callable:int} dict, the task to be performed will be picked randomly, but each task will be weighted according to its corresponding int value. So in the following case, ThreadPage will be fifteen times more likely to be picked than write_post:

class ForumPage(TaskSet):
    tasks = {ThreadPage:15, write_post:1}
